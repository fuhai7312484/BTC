# Polymarket BTC 5m/15m 实时交易信号与概率判断系统计划

## 1. 项目目标

构建一个面向 Polymarket 的 BTC 实时分析系统，能够围绕 5 分钟和 15 分钟两个交易周期，持续输出：

1. 方向性交易信号：看涨 / 看跌 / 观望
2. 概率判断：未来 5m、15m 内价格方向或目标区间命中的概率
3. 实时反馈：随着市场数据变化，动态刷新信号和概率
4. 可解释分析：展示触发信号的核心因子，避免只给黑盒结论

核心要求是“实时获取分析数据”，因此系统不能依赖离线批处理结果，而要以流式采集、增量计算和低延迟推理为主。

---

## 2. 业务定义

### 2.1 分析对象

- 标的：BTC
- 周期：5 分钟、15 分钟
- 输出频率：当前系统按 1 秒刷新一次，后续接入 WebSocket 后可基于行情变化事件触发刷新
- 输出形式：
  - 交易信号：Long / Short / Neutral
  - 概率值：0-100%
  - 置信度：高 / 中 / 低
  - 关键因子：动量、波动率、盘口强弱、Polymarket 价格偏离、外部现货/永续价格驱动等

### 2.2 信号定义

建议将信号拆成两层：

- 第一层：方向判断
  - 看涨：短周期动量与概率模型共同支持上涨
  - 看跌：短周期动量与概率模型共同支持下跌
  - 观望：市场噪声大、分歧高、胜率不稳定
- 第二层：概率判断
  - 未来 5m 上涨概率
  - 未来 15m 上涨概率
  - 价格触达某区间的概率

## 3. 数据源设计

系统必须实时获取多源数据，建议至少包含以下几类：

### 3.1 Polymarket 数据

- 相关 BTC 市场价格
- 市场深度、成交量、订单簿变化
- 价格变化速度、买卖盘倾斜
- 若有可用接口，实时监听 market updates / trades / order book events

#### 3.1.1 可直接从接口或市场元数据拿到的字段

- `current_price`：Polymarket 合约当前交易价格，通常可用 best bid / best ask / mid / last trade 表示
- `market_id` / `question` / `slug`：用于识别 5m、15m 对应的具体市场
- `outcomes`：市场结果选项，例如 Yes / No 或区间选项
- `rules` / `description`：用于解析该市场的定价逻辑和结算条件
- `start_time` / `end_time`：用于定位 5 分钟或 15 分钟合约的时间边界
- `priceToBeat`：BTC Up/Down 事件可能由 Gamma 事件对象直接提供该窗口的起始参考价，应优先作为目标价
- `orderbook`：买一卖一、深度、挂单分布
- `trades`：最新成交价、成交量、成交方向

#### 3.1.2 需要系统实时计算的字段

- `target_price`：优先读取 Gamma 的 `priceToBeat`；没有该字段时再从显式配置、市场标题、描述或规则中解析
- `price_gap`：当前 BTC 现货价与目标价格的差值，或 Polymarket 合约隐含价格与模型公允价的偏离
- `implied_probability`：由 Polymarket 当前价格映射得到的隐含概率
- `distance_to_target`：现价距离目标价的百分比或绝对值差
- `fair_value_gap`：当前合约价格与模型推导公允价之间的偏差

- 说明：Polymarket 接口更擅长提供“合约当前价格”和“市场状态”，而“目标价格”通常是市场定义的一部分，需要系统在接入层解析并标准化。

### 3.2 BTC 外部行情数据

- 现货交易所：Binance、Coinbase、OKX 等
- 永续合约：资金费率、持仓量、爆仓数据、基差
- 盘口数据：best bid/ask、买卖盘深度、成交强度

### 3.3 衍生辅助数据

- BTC 波动率指标
- 相关宏观事件日历
- 新闻情绪或突发事件标签
- 链上或大资金异动指标（可作为二期）

### 3.4 数据刷新策略

- 行情订阅优先使用 WebSocket 或流式接口
- 兜底使用 REST polling
- 对关键字段做时间戳对齐，统一到事件时间轴
- 为每个信号周期维护滑动窗口：例如 5m 窗口、15m 窗口

## 4. 指标与特征设计

### 4.1 5 分钟特征

- 最近 30s / 1m / 3m / 5m 收益率
- 价格加速度与斜率
- 成交量突增比
- 买卖盘失衡度
- Polymarket 价格相对外部现货价格的偏离
- 短期波动率与振幅

### 4.2 15 分钟特征

- 最近 5m / 10m / 15m 趋势强度
- 趋势持续性
- 波动率收敛或扩张
- 分时成交密度
- 多源价格协同性
- 更大级别阻力 / 支撑附近的反应

### 4.3 结构化特征输出

建议统一生成标准特征表：

- `timestamp`
- `target_price`
- `current_price`
- `price_gap`
- `market_price`
- `spot_price`
- `perp_price`
- `spread`
- `order_imbalance`
- `volume_delta`
- `return_1m`
- `return_5m`
- `return_15m`
- `volatility_5m`
- `volatility_15m`
- `signal_score_5m`
- `signal_score_15m`

## 5. 概率判断方法

### 5.1 基础方案

先用可解释、可快速上线的方案：

- 规则引擎 + 统计模型结合
- 输出上涨概率、下跌概率、观望概率
- 根据历史回测校准概率分布

### 5.2 进阶方案

- 分类模型：LightGBM / XGBoost / CatBoost
- 时间序列模型：LSTM / Temporal Convolution / Transformer 类模型
- 概率校准：Platt Scaling、Isotonic Regression
- 置信度修正：根据当前波动率、流动性、异常行情进行惩罚或加权

### 5.3 目标标签定义

建议分别定义 5m 和 15m 标签：

- 5m 标签：未来 5 分钟后 BTC 价格相对当前的方向
- 15m 标签：未来 15 分钟后 BTC 价格相对当前的方向
- 可选增强：
  - 是否突破上方阈值
  - 是否跌破下方阈值
  - 是否进入高波动区间

## 6. 实时系统架构

### 6.1 数据层

- 原始数据存储
- 特征计算服务

### 6.2 计算层

- 实时特征引擎
- 信号计算引擎

### 6.3 展示层

- Web Dashboard

### 6.4 推荐技术形态

## 7. 信号计算逻辑

### 7.1 多层打分

建议构建 0-100 的综合评分：

- 趋势分
- 量能分
- 盘口分
- 跨市场一致性分
- 异常波动惩罚分

最终映射为：

- 80-100：强看涨 / 强看跌
- 60-79：偏多 / 偏空
- 40-59：观望
- 0-39：反向弱势

### 7.2 周期拆分

- 5m 信号更偏向高频动量、盘口、成交强度
- 15m 信号更偏向趋势持续性、波动结构和跨市场确认

### 7.3 风险过滤

在以下场景降低信号强度：

- 极端跳涨跳跌
- 低流动性
- Polymarket 与外部市场明显背离但无确认
- 数据源延迟或缺失

---

## 8. 实时性要求

### 8.1 延迟目标

- 数据进入系统到页面可见：建议 1-3 秒内
- 信号更新延迟：建议 3-10 秒内
- 模型推理：单次最好低于 200ms

### 8.2 容错策略

- 数据源断流自动切换备用源
- 价格异常值过滤
- 缺失字段时使用最近一次有效值或降级策略
- 输出明确标注“数据延迟 / 数据不完整”状态

---

## 9. 验证与回测

### 9.1 离线回测

- 用历史 BTC 行情和 Polymarket 价格回放验证
- 统计胜率、收益分布、最大回撤、误报率
- 分别评估 5m 与 15m 表现

### 9.2 在线验证

- 实盘模拟模式先运行
- 记录每次信号与后续结果的偏差
- 持续做概率校准

### 9.3 核心指标

- 准确率
- AUC / F1
- 概率校准误差
- 信号触发后 5m / 15m 的方向命中率
- 平均响应延迟

## 11. 风险点

- Polymarket 数据接口稳定性不确定
- BTC 短周期噪声高，5m 信号容易过拟合
- 多源数据不同步会造成错误判断
- 概率输出需要校准，否则容易“看起来很准，实际很飘”
- 若没有严格回测，实时信号容易失真

## 14. 当前实现状态

当前仓库已完成第一阶段原型的主要链路：

- 默认提供 BTC 5m / 15m 两个市场定义，并支持仿真数据持续刷新。
- live 模式自动通过 Gamma API 搜索 BTC Up/Down 5m、15m 市场，解析 `clobTokenIds`、规则中的目标价和 Up/Down 方向。
- 通过 Polymarket CLOB REST 接口读取 Up token 的订单簿、midpoint 和最新成交价。
- 通过 Binance spot / perpetual REST 接口读取当前 BTC 价格、盘口和成交额字段。
- 标准化输出 `current_price`、`target_price`、`price_gap`、`contract_price`、`implied_probability`，并标明数据源模式。
- 标准化输出 UP/DOWN 合约的实时买入价与卖出价，并在页面使用盘口表格展示。
- UP/DOWN 买卖价优先使用 CLOB `/price` 的 `BUY` / `SELL` 可成交报价，订单簿和 Gamma 仅作为带来源标记的降级数据。
- 所有可读时间统一为 `yyyy-MM-DD hh:mm:ss`，5m/15m 面板按各自市场结束时间显示 `mm:ss` 倒计时。
- 使用 REST polling + SSE 页面推送完成实时反馈；默认轮询间隔为 1 秒。
- 已加入解析、CLOB 字段映射、市场详情、5m/15m 信号刷新测试。

以下内容仍属于后续阶段，不应把当前概率当作经过校准的投资建议：

- Polymarket Market Channel WebSocket 增量订单簿与断线重连。
- 历史行情和信号落库、回放、回测及收益/回撤统计。
- 真实历史样本上的概率校准和模型训练。
- 告警、权限、审计、监控及生产级多源容灾。
